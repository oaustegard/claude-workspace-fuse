# Grok Build — Agent Runtime Cluster Inventory

## 1. Architecture

**Entry points** (`xai-grok-shell/src/agent/app.rs`): `run_stdio_agent` (:289), `run_headless`/`run_headless_no_browser` (:409/:419), `run_leader` (:917). All three ultimately construct an `MvpAgent` (`agent/mvp_agent/acp_agent.rs`) that implements the `agent_client_protocol` (ACP) `Agent` trait — same protocol family Zed uses. `MvpAgent::initialize` (mvp_agent/acp_agent.rs:21) starts the subagent coordinator, cleans stale worktree-pool entries, bootstraps the search index, and migrates legacy trust state.

**Leader/follower daemon model** (`leader/mod.rs:1-33`, doc comment): a single **leader process per machine** owns the `MvpAgent` and all session state, persisting to `~/.grok/`. TUI, IDE-extension, and headless CLI clients are thin front-ends that connect over a Unix socket (`~/.grok/leader.sock`) using framed ACP JSON (`leader/protocol.rs: read_frame/write_frame/read_message/write_message`). `connect_or_spawn` (leader/mod.rs:1205) either attaches to a running leader or spawns one; version comparison (`leader_is_older_than`, `should_evict`) makes newer clients evict older leaders (never the reverse). This lets multiple UIs share one live agent/session set — analogous to a persistent language-server daemon rather than a per-invocation CLI process.

**Turn/session actor loop**: each session is a `SessionActor` driven by `run_session` (`session/acp_session_impl/run_loop.rs:33-820`), a single `tokio::select!` biased loop multiplexing: idle-memory-flush timer, memory-"dream" timer, model-switch notifications (resets laziness state), chat-state events (context-window pruning, image-budget eviction), buffered replay/session-update events, turn-completion results (`completion_rx`, drives goal-degradation handling + post-turn laziness check), and inbound `SessionCommand`s (prompts, cancels, hooks). Turns are spawned via `spawn_session_actor`/`spawn_session_on_thread` (`session/acp_session_impl/spawn.rs:91,1688`) with an enormous parameter surface (200+ args) reflecting how much per-session config (models, MCP servers, plugins, goal/subagent toggles, feedback, worktree, permission handles) is threaded through.

**Tool dispatch**: `dispatch_tool` (`session/acp_session_impl/tool_dispatch.rs:13`) resolves a `PreparedToolCall` against `WorkspaceOps`, producing a `ToolRunResult`/`ToolError` (`xai-tool-runtime`/`xai-tool-protocol`). Plan-mode edit gating (`plan_mode_edit_gate`, tool_calls.rs:166) and lock-path extraction (`lock_path_for_args`, tool_dispatch.rs:56) guard concurrent/blocked tool calls. Streaming tool output uses `ToolStream`/`with_progress`/`terminal_only` (`xai-tool-runtime/src/tool.rs:206,218`) and `stream_chunk` (`streaming.rs:85`) for incremental progress frames.

**Subagent spawning**: `handle_subagent_request` (`agent/subagent/handle_request.rs:59`) runs against a `SubagentCoordinator`; `reconcile_orphaned_subagents` (`agent/subagent/mod.rs:2731`) recovers unfinished children after a crash/restart. Subagents get their own worktree from a pooled allocator (`session/worktree_pool.rs`, `should_enable_pool` :1107) so parallel subagents don't collide on the working tree.

**Session/state model**: conversation state lives in `xai-chat-state` as an actor (`actor/mod.rs`, `actor/state.rs`) with token-estimation-driven pruning (`should_prune`, `prune_conversation` — request_builder.rs:158,166), image-byte-budget eviction (`compact_images_to_byte_budget`:395), and a two-tier compaction system (`compaction_utils.rs`, `compaction_transcript.rs` — segments summaries to numbered markdown files with extracted keywords for later retrieval). Persistence is JSONL-based (`session/storage/jsonl`, `session/storage/mod.rs`) with rewind-line filtering, replay-from-cursor, and remote sync of the search index to GCS (`storage/search_remote_sync.rs`).

## 2. Key abstractions

- **`GoalOrchestration`** (`session/goal_tracker.rs`, shape via `make_base_orchestration`:1339) — the state for the autonomous "goal harness": `GoalPhase{Idle,Planning,Executing}`, `GoalStatus` (Active + 6 paused variants: `UserPaused`, `BackOffPaused`, `NoProgressPaused`, `InfraPaused`, `Blocked`, ...), token budget/high-water tracking, verifier bookkeeping (`skeptic0_session_id`, `last_gap_fingerprint`, `classifier_stall_count`, `consecutive_not_achieved`), and live per-turn telemetry.
- **`SessionActor`** — the per-session actor object driving `run_session`; owns `chat_state_handle`, `mcp_state`, `memory`, `models_manager`, goal/laziness/plan-mode trackers.
- **`MvpAgent`** (`agent/mvp_agent/acp_agent.rs`) — the top-level ACP `Agent` impl; one per leader process, fans requests out to per-session actors.
- **`ToolBridge`/`ToolRunResult`/`ToolError`** (`xai-tool-runtime`, `xai-tool-protocol`) — uniform tool execution/streaming/error envelope shared by local tools, MCP tools, and backend-hosted tools.
- **`AcpGatewaySender`/`AcpAgentChannel`** (`xai-acp-lib/src/gateway.rs:99`, `channel.rs:28`) — typed request/response channel wrapping ACP JSON-RPC over the leader socket or stdio.
- **`GoalClassifierSpawner`/`GoalPlannerSpawner`/`GoalStrategistSpawner`** (goal_classifier.rs, goal_planner.rs, goal_strategist.rs) — trait-abstracted spawners for the planner/implementer/skeptic/strategist "roles," each a full sub-conversation spawned like a subagent.
- **`ChatStateHandle`** (`xai-chat-state`) — actor handle for conversation mutation, pruning, and compaction, decoupled from the session actor.
- **`WorkspaceOps`** (`xai_grok_workspace`, used throughout) — filesystem/permission/git abstraction gating tool execution (plan-mode, worktrees, permission rules).
- **`LeaderClient`/`LeaderLock`/`ClientMode`** (`leader/`) — daemon connection/registration/lock primitives implementing the multi-client-single-agent model.

## 3. Notable/novel techniques

- **Adversarial multi-skeptic verification panel** (`goal_classifier.rs: run_verification_stage`, :1949-2295). After an implementer subagent claims a goal is done, a **persistent "skeptic 0" gatekeeper** re-reviews the diff first (cold on attempt 1, *resumed* on later attempts so it keeps memory of prior rejections). A high-confidence, non-blocking refute from skeptic 0 alone short-circuits (`Achieved` is impossible without full-panel override). Otherwise it fans out `N` parallel skeptics (`futures::future::join_all`) over a **shared, once-captured** git diff (`capture_changes_diff`) and plan-diff, aggregated by quorum (`aggregate_skeptic_verdicts`). Everything fails open (`record_fail_open`) on path-validation or I/O errors rather than blocking the goal.
- **Idle-stall / laziness classifier with a nudge budget** (`session/acp_session_impl/laziness_classifier.rs`). A separate LLM call classifies the live transcript into categories (`StalledNarration`, `StalledPermissionAsking`, `StalledNoTodosButTaskInFlight`, `StalledFalseCompletion`) and only injects a corrective system reminder (`build_laziness_nudge`) if confidence exceeds a per-model threshold **and** a per-session nudge cap hasn't been exhausted (`evaluate_laziness`:781) — i.e., rate-limited automatic steering rather than a single static reminder.
- **Goal-harness multi-role loop**: planner → implementer → skeptic-panel → strategist (fires every-N rounds, `strategist_should_fire`:102) → summarizer, each a distinct spawned sub-conversation with its own scratch dir (`implementer_scratch_dir`, `skeptic_scratch_dir`) and its own "fail-open retry" wrapper (`spawn_with_fail_open_retry`, goal_planner.rs:132) so infra flakiness degrades gracefully instead of aborting the goal.
- **Stop-word/pattern detection over model output** (`goal_stop_detector.rs: matched_stop_pattern`) to catch a subagent claiming it's blocked/done in free text, cross-checked against the classifier verdict.
- **Two-pass compaction** (`session/two_pass.rs`) — splits conversation at a fraction, summarizes pass 1, feeds a "note" of the pass-1 raw output into pass 2's prompt (`note_for_two_pass_pass2`) for compaction quality without a single giant summarization call.
- **Todo-gate enforcement** (`acp_session_impl/reminders.rs: evaluate_todo_gate/build_todo_gate_reminder`) — detects pending/unbacked in-progress todos and injects a reminder rather than relying purely on prompt instructions.
- **Memory "dream" and idle flush** — background timers (`dream_check_sleep`, `idle_flush_sleep` in run_loop.rs) periodically extract durable memories from idle conversations (`run_memory_flush`, `session::memory::hooks::on_session_end`) with semantic-duplicate suppression (`is_semantically_duplicate` using an `EmbeddingProvider`).
- **Prompt-offload-to-file** (`prompt_build.rs: write_offload_and_build/build_truncated_prompt_message`) — oversized user messages get written to a scratch file with a truncated head/tail shown inline, avoiding context blowup while preserving full content on disk.
- **Worktree pooling for parallel tool/subagent execution** (`worktree_pool.rs`) — pre-warmed git worktrees adopted by new subagents (`should_enable_pool`, `take_adoptable_worktrees`) rather than cloning per spawn.
- **Server-fetched subagent/tool bundles** (`remote/client.rs: fetch_subagent_bundle`, `fetch_bundle`) — subagent type definitions and model catalogs can be pulled from the backend at runtime, letting the vendor push new subagent personas without a client release.

## 4. Useful aspects to borrow (skills/hooks/orchestration/memory)

- **Escalating skeptic panel pattern**: cheap single-verifier-first, only fan out to a full adversarial panel on ambiguous verdicts, with one verifier kept "warm" (resumed) across retries to accumulate rejection memory. Directly portable to a Claude Code skill-verification loop (e.g., a `/verify` skill that resumes the same skeptic subagent across fix attempts instead of spawning fresh, amnesiac reviewers each time).
- **Confidence + budget-gated nudging** instead of always-on reminders — apply the same pattern to hook-based steering (e.g., a Stop-hook or PostToolUse hook that only fires a correction after classifying stall type and confidence, capped per session) to avoid reminder fatigue.
- **Fail-open everywhere in the verification/orchestration path**: every scratch-file write, diff capture, and role-spawn has an explicit fail-open branch that degrades to "skip check" rather than blocking user progress — a good default for building hook/verification infrastructure that must not become a productivity blocker.
- **Todo/plan gate as structured state, not just prompt text**: pending/in-progress todo detection is done programmatically (`evaluate_todo_gate`) and only then surfaced as a reminder — worth mirroring for a memory/orchestration system that wants reliable "did the agent actually finish the plan" signals rather than trusting free-text claims.
- **Session-end memory hook with semantic dedup** (`memory/hooks.rs`, `memory_flush.rs: is_semantically_duplicate`) is a clean minimal design for an auto-memory system: flush on idle timer or session end, embed + threshold-compare against existing memory index before writing, avoids duplicate memory buildup.
- **Goal-scratch directories per verifier/attempt** (`goal_scratch_root`, `implementer_scratch_dir`, `skeptic_scratch_dir`) give each role/attempt an isolated, addressable filesystem workspace for artifacts (verdicts, diffs, details) — a reusable convention for any multi-agent orchestration needing auditable per-role state.
- **Leader/daemon model** for a memory or orchestration system: one long-lived process owning state/persistence, multiple thin UI clients attaching via socket with version-based eviction — useful if building cross-tool (CLI + IDE + web) shared session/memory access.
- **Compaction-transcript segmenting to keyword-indexed markdown files** (`compaction_transcript.rs`) is a nice retrieval-friendly compaction format (`render_segment_md` + `extract_keywords` + index row) worth borrowing for a memory system's "old conversation" archive layer.
